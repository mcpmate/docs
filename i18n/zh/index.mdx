## MCP Server 使用痛点

- **重复设置** – 每个 MCP 兼容的编辑器（Cursor、Claude Desktop、Zed…）必须单独配置。
- **繁琐切换** – 切换项目或工作流程意味着启动/停止服务器和手动编辑配置。
- **资源浪费** – 运行多个服务器实例会消耗 CPU 和内存。

## 传统模式 vs MCPMate 加持

| 比较维度 | 传统设置 | MCPMate |
| ------ | ---------------- | ------------ |
| 配置时间 | 每个应用高 | 低（单个位置） |
| 系统资源   | 高           | 优化代理 |
| 工作流程切换    | 手动编辑   | 一键切换 |
