## 什么是配置？

配置是一个命名的集合，包含：
- MCP 服务器及其设置
- 启用的工具及其优先级
- 提示库和模板
- 资源访问权限

将配置视为可以即时切换的预设。

## 创建自定义配置

<Tabs>
  <Tab label="从头开始">
    1. 在配置面板中点击 **新建配置**
    2. 给它一个描述性名称（例如，"数据分析"）
    3. 从可用列表中添加服务器
    4. 配置工具优先级和权限
    5. 保存并激活
  </Tab>
  
  <Tab label="克隆现有">
    1. 选择一个现有配置作为模板
    2. 点击 **克隆** 并给它一个新名称
    3. 根据需要修改服务器、工具或设置
    4. 保存您的自定义版本
  </Tab>
</Tabs>

## 切换配置

<Callout type="info">
  切换配置立即生效—无需重启客户端应用。
</Callout>

切换活动配置：
1. 打开配置面板
2. 点击您想要切换的配置旁边的切换按钮
3. 等待 "Active" 指示器出现
4. 您的客户端应用现在可以访问新的工具集

## 最佳实践

- **描述性名称**：使用清晰、工作流程特定的名称
- **最小化配置**：仅包含必要的服务器以减少资源使用
- **定期清理**：删除未使用或过时的配置
