## Adding Servers

<Tabs>
  <Tab label="From Registry">
    1. Click **Add Server** in the Servers panel
    2. Browse the built-in server registry
    3. Select a server and click **Install**
    4. Configure any required settings (API keys, endpoints)
    5. Test the connection and save
  </Tab>
  
  <Tab label="Custom Server">
    1. Choose **Add Custom Server**
    2. Specify connection details:
       - **Name**: Descriptive server name
       - **Type**: stdio, SSE, or HTTP
       - **Command/URL**: Execution command or endpoint
       - **Environment**: Required environment variables
    3. Test and save the configuration
  </Tab>
</Tabs>

## Server Status

Each server shows:
- **Health indicator**: Green (healthy), Yellow (warning), Red (error)
- **Last activity**: When the server was last used
- **Tool count**: Number of available tools
- **Resource usage**: CPU and memory consumption

<Callout type="info">
  Servers are started on-demand and automatically stopped when not in use to save resources.
</Callout>

## Server Configuration

  📸 Server configuration dialog screenshot placeholder
</div>

### Connection Settings
- **Protocol**: Choose between stdio, SSE, or HTTP
- **Timeout**: Request timeout in seconds
- **Retry policy**: Automatic retry on failure
- **Rate limiting**: Requests per second limits

### Environment Variables
- **API Keys**: Secure storage for authentication
- **Endpoints**: Custom server URLs
- **Debug flags**: Enable detailed logging

### Resource Limits
- **Memory limit**: Maximum RAM usage
- **CPU throttling**: Prevent excessive CPU usage
- **Concurrent requests**: Limit parallel operations

## Troubleshooting

Common server issues:

- **Connection failed**: Check network settings and credentials
- **High resource usage**: Review server configuration and limits
- **Tools not appearing**: Verify server is enabled in active profile
- **Timeout errors**: Increase timeout values or check server performance
